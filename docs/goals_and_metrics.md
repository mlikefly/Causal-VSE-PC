# Causal-VSE-PC 目标与指标

**文档版本**: 1.0  
**创建日期**: 2025年12月  
**核心创新**: 因果推断驱动的加密策略

---

## 一、总体目标

### 1.1 核心目标

Causal-VSE-PC旨在实现**基于因果推断的隐私保护与任务可用性平衡**，具体目标：

1. **隐私保护**：防止身份识别、属性推断、图像重建
2. **任务可用性**：在加密图像上直接运行ML任务，无需解密
3. **可解释性**：基于因果推断的可解释加密策略
4. **可验证性**：轻量级Hash Commitment验证加密正确性
5. **高效性**：加密速度 > 10 FPS，满足实时应用需求

### 1.2 核心创新目标

```
1. 因果推断驱动的加密策略（单点突破）
   - 首次提出基于SCM的隐私预算分配理论
   - 量化加密对ML性能的因果效应（ATE/CATE）
   - 生成可解释的加密策略报告

2. 轻量级可验证性（系统亮点）
   - Hash Commitment + HMAC（简化实现）
   - 无需zk-SNARKs的复杂实现

3. 密文域ML适配器（方法创新）
   - 标准ML模型在加密图像上直接工作
   - 无需解密，保持性能
```

### 1.3 应用场景目标

- **医疗图像隐私计算**：医院加密患者图像，云端AI直接分析，可解释加密策略
- **人脸识别隐私保护**：加密人脸图像，识别系统在密文上工作，因果驱动的隐私保护
- **自动驾驶数据共享**：车辆加密图像，云端分析交通情况，任务可用性保障

## 三、因果推断相关指标（核心创新）

### 3.1 因果效应指标

```
1. ATE (Average Treatment Effect)
   - 定义: E[Y(high_privacy) - Y(low_privacy)]
   - 目标: 
     ├─ 识别任务: ATE < -0.90 (识别率下降 > 90%)
     ├─ 分类任务: ATE > -0.20 (分类准确率下降 < 20%)
     └─ 分割任务: ATE > -0.30 (mIoU下降 < 30%)
   - 当前: ⚠️ 待验证

2. CATE (Conditional ATE)
   - 定义: E[Y(high) - Y(low) | X=sensitive]
   - 目标:
     ├─ 敏感区域: CATE < -0.95 (隐私保护强)
     └─ 任务区域: CATE > -0.10 (任务可用性保留)
   - 当前: ⚠️ 待验证

3. 因果解释准确度
   - 定义: 因果报告与实际效应的匹配度
   - 目标: > 90%
   - 评估: 人工评估 + 自动评估
   - 当前: ⚠️ 待验证
```

### 3.2 可解释性指标

```
1. 因果报告完整性
   - 定义: 报告包含所有关键信息（ATE, CATE, 解释）
   - 目标: 100%
   - 当前: ⚠️ 待实现

2. 自然语言解释质量
   - 定义: 人类可理解度
   - 目标: > 85% (人工评估)
   - 评估: 5名评估者评分
   - 当前: ⚠️ 待验证

3. 因果图可视化质量
   - 定义: 因果图的可读性和准确性
   - 目标: 清晰、准确
   - 当前: ⚠️ 待实现
```

## 四、具体目标分解

### 4.1 理论目标

#### 目标1：因果推断理论框架

```
内容：
1. 结构因果模型（SCM）定义
   - 节点: X (语义), T (任务), Z (隐私预算), Y (性能)
   - 边: 因果关系定义
   
2. ATE/CATE计算公式
   - ATE = E[Y(1) - Y(0)]
   - CATE = E[Y(1) - Y(0) | X=x]
   
3. 因果识别条件
   - 无混淆假设
   - 一致性假设
   - 正性假设

时间：1个月
优先级：P0
```

#### 目标2：可解释性理论

```
内容：
1. 因果解释的生成方法
2. 自然语言解释的生成规则
3. 因果报告的结构化定义

时间：2周
优先级：P0
```

### 4.2 实验目标

数据集：CelebA-HQ, CelebAMask-HQ
时间：1个月

预期结果：
✅ ATE识别 < -0.90 (隐私保护强)
✅ ATE分类 > -0.20 (任务可用性保留)

预期结果：

预期结果：

✅ 优于同态加密和安全多方计算
